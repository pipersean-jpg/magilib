## What Is MagiLib?
A Progressive Web App (PWA) for cataloguing magic book collections. Users can add books by title, attach cover images, track condition and pricing, and view collection stats. Built with vanilla JavaScript — no framework. Hosted on Vercel, backed by Supabase.

## Architecture

### Frontend
Pure HTML + CSS + vanilla JS. No build step, no bundler. index.html is HTML/CSS only (111 KB). All JS loaded via script tags in index.html.

### Backend — Supabase (Postgres)
- **books** — main collection, RLS on (user_id = auth.uid())
- **profiles** — one row per user, linked to auth.users
- **covers** — cover image metadata (empty, reserved for future use)
- **price_db** — price cache (empty, reserved for future use)
- **admin_users** — ⚠️ RLS is OFF — security advisory open

### Proxy APIs — Vercel Serverless Functions
- api/fetch-proxy.js — proxies external HTTP fetches (CORS workaround for cover images)
- api/claude-proxy.js — proxies Anthropic API calls for AI-powered price lookups

### Local Price / Metadata Databases (JS files)
- conjuring_db.js — 3,499 title→cover URL mappings from the Conjuring Archive
- magilib_price_db.js — Murphys Magic in-print MSRPs
- magilib_disc_db.js — discontinued title list
- magilib_market_db.js — secondary market prices (eBay/QTTE/CMB/MC)

## Challenges Faced & How They Were Solved

**1. Monolithic File / Context Window Overflow**  
The original app was a single index.html with all JS inline (~111 KB). Loading it consumed most of Claude's context window, making edits unreliable.  
Solution: JS was split into 7 separate files in April 2026. Each session now loads only the one relevant file.

**2. Supabase Client Blocked by CSP**  
Vercel's Content Security Policy blocked the Supabase JS client from loading off the CDN (cdn.jsdelivr.net).  
Solution: supabase.min.js is self-hosted in the repo root. vercel.json CSP headers updated accordingly.

**3. Legacy Google Sheets Architecture**  
The app originally used Google Apps Script + Google Sheets as its backend. The full migration to Supabase is complete, but legacy references remain visible in the wizard panel, FAQ answers, and support diagnostics.  
Status: Functional, but cleanup is pending.

**4. No-Code Deploy Workflow**  
The user has no experience with terminals, GitHub CLI, or dev tools.  
Solution: A browser deploy artifact was built that wraps the GitHub Contents API. User pastes code and clicks Deploy.

**5. Supabase Init Timing Errors**  
Files were referencing _supa and _supaUser at parse time before DOMContentLoaded fired, causing null reference errors on load.  
Solution: Both variables are null at parse time and only assigned inside DOMContentLoaded in ui.js. All other files reference them as globals but never initialise them.

**6. Cover Images**  
External cover image URLs expired or were CORS-blocked, leaving books with missing covers.  
Solution: conjuring_db.js provides 3,499 local title→cover mappings. api/fetch-proxy.js handles external fetches that would otherwise be CORS-blocked.
